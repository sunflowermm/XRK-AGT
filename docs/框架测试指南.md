# 框架测试指南

> 基准环境：Windows 10，Node v24.12，pnpm 10.32，2026-05-17 实测。  
> **框架成绩只统计 `core/system-Core/` 官方自带模块**；扩展 core 与个人后期文件**不计入**。

## 测试口径

| 范围 | 是否计入框架测试 | 说明 |
|------|------------------|------|
| `core/system-Core/` 自带模块 | ✅ 是 | 见下表基准数量 |
| `core/*` 其他目录 | ❌ 否 | 可选扩展 core |
| 后期个人文件 | ❌ 否 | 如已移除的 `screen.js`、`react-bits-mcp.js`、`远行商人.js`；若放回目录，测试亦通过 `SYSTEM_CORE_NON_BASELINE` 排除 |

**system-Core 基准规模（静态 + 加载器断言）**

| 类型 | 数量 | 说明 |
|------|------|------|
| HTTP API | **11** | |
| AI 工作流 (stream) | **7** | chat、desktop、tools、memory、database、web、browser |
| 插件 (plugin) | **15** | 不含「远行商人」等个人插件 |
| Tasker | **4** | |
| events | **3** | |

扩展 core 或误放回的个人文件存在时，运行时日志可能更大（例如全仓库 HTTP 19、插件 22、工作流 13），**仅作运维参考，不作为框架达标线**。

## 命令

| 命令 | 用途 | 典型耗时 |
|------|------|----------|
| `pnpm test` | CI 默认：结构 + 配置 + system-Core 加载器断言 | ~3s，**60 项** |
| `pnpm test:e2e` | 子进程真启动 Bot，探活 HTTP | ~8s，**3 项** |
| `pnpm test:all` | 含 e2e（Windows 退出偶发 libuv 提示） | ~12s |
| `pnpm test:fast` | 跳过加载器集成 | ~2s |

## 与常见工程实践的对应

| 维度 | 常见做法（Nest / Spring Boot / 大型 Node 网关） | XRK-AGT 对应 | 可验证结果 |
|------|-----------------------------------------------|--------------|------------|
| 分层与边界 | 基础设施与业务分包，业务可插拔 | `src/` 基类 + `core/<name>/` | 业务不在 `src/` |
| 契约 / 结构测试 | 路由、模块清单回归 | `http-api-structure`、`module-inventory` | 11 HTTP 结构稳定 |
| 配置治理 | 默认配置 + Schema | `config-alignment`（20 项） | YAML 与 schema 段一致 |
| 安全基线 | 鉴权、路径校验 | `auth-loopback`、`input-validator` | `/api/*` 默认 Key |
| 重依赖懒加载 | 浏览器客户端延后 | `renderer-lazy` | 单测不拉起 Playwright |
| 冒烟 + E2E | 真进程 + 关键 API | `server-e2e`（3 项） | `/xrk/`、`/api/plugins/summary` |
| 测试环境隔离 | 跳过远程 MCP | `XRK_TEST=1` | 集成测不阻塞 |

**框架优越性（可复述要点）**

1. **基准不含个人扩展**：`11+7+15+4+3` 仅锁官方自带模块；screen / react-bits-mcp / 远行商人等不计入。  
2. **扩展 core 不稀释指标**：第三方 core 自由增减，不改动框架达标线。  
3. **启动与 CI 更轻**：渲染器懒加载、测试跳过远程 MCP。  
4. **配置与安全可测**：三件套对齐 + 鉴权/路径单测。
