# XRK-AGT 框架测试指南

> 面向发布前自检、CI 与人工回归。数量与路径以**当前代码**为准（2026-05-17）。

## 1. 测试目标

| 维度 | 说明 |
|------|------|
| **框架完整性** | 加载器能扫描 `core/*/` 下 plugin/http/stream/tasker/events/commonconfig |
| **配置利用率** | YAML 模板 ↔ `commonconfig` schema ↔ `cfg.*` 消费一致，避免「只配不用」 |
| **资源利用** | `monitor` 驱动 `SystemMonitor`；`redis`/`mongodb` 读 `options`；静态资源压缩/缓存按 `server` |
| **安全基线** | `/api/*` 默认鉴权、`127.*` 回环例外、路径校验 |
| **控制台** | `core/system-Core/www/xrk/` 四页可用、空态/错误态/加载态完整 |

## 2. 自动化冒烟（推荐每次改框架后执行）

```bash
pnpm test
```

等价于：

```bash
node --test tests/framework/*.test.mjs
```

覆盖项：

- 全局/端口配置均有 `config/default_config/<name>.yaml`
- `system.js` schema 包含 `GLOBAL_CONFIGS` + `SERVER_CONFIGS` 各段
- system-Core：**11** 个 HTTP、**9** 个工作流（含 `screen`、`react-bits-mcp`）
- `www/xrk` 关键静态资源存在
- `ui-kit` 情绪 key 白名单

## 3. 启动与端到端冒烟

### 3.1 启动

```bash
pnpm install
node app
```

期望：无未捕获异常；日志出现 HTTP 监听端口；Tasker/插件/工作流加载完成。

### 3.2 Web 控制台（`/xrk`）

| 步骤 | 操作 | 期望 |
|------|------|------|
| 1 | 浏览器打开 `http://127.0.0.1:<port>/xrk` | 首页「系统概览」加载 |
| 2 | 查看统计卡片与 CPU/内存/网络图 | 数值或占位 `--`，图表无报错 |
| 3 | 插件与工作流词云 | 有数据则芯片可悬停详情；无数据有空态文案 |
| 4 | 切换「AI 对话 / 配置管理 / API 调试」 | 标题与侧栏高亮正确 |
| 5 | 配置页保存一项 | 写入 `data/server_bots/{port}/` 对应 yaml |
| 6 | API 调试调用 `/api/plugins/summary` | 本机回环可不带 Key；外网需 Key |

### 3.3 HTTP API 抽样

鉴权规则见 [AUTH.md](AUTH.md)。本机 `127.0.0.1` 可免 Key 访问 `/api/*`（回环例外）。

```bash
# 插件摘要（需服务已启动）
curl -s http://127.0.0.1:PORT/api/plugins/summary | head -c 200

# 通知测试（需 notice.yaml 配好渠道，且为 POST）
curl -s -X POST http://127.0.0.1:PORT/api/notice/test \
  -H "Content-Type: application/json" \
  -d "{\"title\":\"test\",\"content\":\"framework test\"}"
```

## 4. 配置消费对照（框架级）

路径常量：`src/infrastructure/config/config-constants.js`

### 4.1 全局配置 `data/server_bots/<name>.yaml`

| 配置 | 主要消费位置 | 说明 |
|------|----------------|------|
| `agt` | `src/infrastructure/log.js`、`plugin/add.js`、`plugin/状态.js` | 日志前缀、`files.*` 路径、`status.showDisk` |
| `device` | `src/utils/aistream-config.js`、`factory/*`、`http/device.js` | ASR/TTS/设备通道 |
| `monitor` | `src/modules/systemmonitor.js`（`cfg.monitor` 启动参数） | 浏览器/内存/CPU/报告 |
| `notice` | `core/system-Core/lib/notice/notice-service.js` | iyuu / sct / feishu_webhook |
| `redis` | `src/infrastructure/redis.js` | `options` 传入客户端 |
| `mongodb` | `src/infrastructure/mongodb.js` | `options` 传入客户端 |

### 4.2 随端口配置 `data/server_bots/{port}/<name>.yaml`

| 配置 | 主要消费位置 |
|------|----------------|
| `server` | `src/bot.js`（压缩、helmet、cors、限流、静态、auth、ws 心跳等） |
| `chatbot` | `core/system-Core/plugin/OneBotEnhancer.js`（黑白名单、私聊、auto） |
| `group` | `plugin/add.js`、`OneBotEnhancer`（冷却、别名、违禁词） |
| `aistream` | `src/infrastructure/aistream/*`、`src/utils/aistream-config.js` |
| `*_llm` | `src/factory/llm/*`、`LLMFactory` |

### 4.3 审计原则

1. **新增字段**：同时改 `config/default_config/`、`commonconfig/*.js`（或 `system.js` 子段）、消费代码。
2. **删除字段**：先确认无 `cfg` 引用，再删 schema，避免控制台仍展示无效项。
3. **已移除**：`server.uiCookie`、扁平 `banned_words_*`、`chatbot.privateChat` 等旧名——勿再写回兼容分支。

详细发布清单见 [代码审查清单.md](代码审查清单.md)。

## 6. 底层/Infrastructure 测试要点

| 模块 | 文件 | 检查项 |
|------|------|--------|
| HttpApi 鉴权 | `src/infrastructure/http/http.js`、`auth.js` | `/api/` 路由自动 `ensureSystemCoreAuth`；回环判定见 `auth.isLoopback127Connection` |

| 输入校验 | `src/utils/input-validator.js` | 文件 API 路径、命令执行白名单 |

## 7. www/xrk 人工检查清单

依据前端技能（`fronted-design`、`interaction-design`、`accessibility-compliance`）：

- [ ] 键盘：`Tab` 可到达侧栏、主按钮；`:focus-visible` 可见
- [ ] 读屏：侧栏 `nav`、活动项 `aria-current="page"`；Toast `aria-live`
- [ ] 加载：`data-updating` 区域有视觉反馈；失败有空态/错误文案
- [ ] 移动：`--vh` 绑定后聊天/全屏无高度跳动
- [ ] 图标：UI 使用 `ui-kit.js` SVG，不用 emoji 作控件图标
- [ ] 情绪：仅 `happy|sad|angry|surprise|love|cool|sleep|think|message`

## 8. 已知缺口与后续

| 项 | 状态 | 建议 |
|----|------|------|
| 单元测试覆盖率 | 低 | 优先为 `auth`、路径校验、配置对齐加 `node:test` |
| E2E 浏览器 | 无 | 可选 Playwright 覆盖 `/xrk` 四页 |
| 配置字段 100% 消费 | 部分 LLM 模板字段仅工厂用 | 用 `grep cfg.<段>` 做增量审计 |
