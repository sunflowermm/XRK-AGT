# 框架测试指南

> 标准值源码：`tests/helpers/system-core.mjs` → `SYSTEM_CORE_BASELINE`。  
> 仅统计 `core/system-Core/`；其他 core 与全仓库加载日志**不计入**达标线。

**文档串联**：[README.md](../README.md) → [docs/README.md](README.md) → [system-core.md](system-core.md) | 发布前：[代码审查清单.md](代码审查清单.md) + [文档审查清单.md](文档审查清单.md)

## 命令

| 命令 | 达标 |
|------|------|
| `pnpm test:fast` | 58 pass，~1.5s |
| `pnpm test` | 63 pass，~5s |
| `pnpm test:e2e` | 3 pass，~8.7s |
| `pnpm test:all` | 66 pass（含 e2e，以当次 CI 日志为准） |

环境：`XRK_TEST=1`（`bootstrap.mjs`）、`node --test --test-force-exit`；E2E 端口 `19000–19799`，就绪信号 `XRK_TEST_READY`。

## 标准值与实测（2026-05-17，Windows 10，Node v24.12）

| 类型 | 标准 | 实测 | 断言 |
|------|------|------|------|
| HTTP | 11 | 11 | `module-inventory` + 11 个 `http/*` key |
| stream | 7 | 7 | 同上 + `streams.has(name)` |
| plugin | 15 | 15 | 同上 + `getPlugins` 过滤 `system-Core` |
| tasker | 4 | 4 | `module-inventory` |
| events | 3 | 3 | `module-inventory` |
| 单测合计（`pnpm test`） | 63 | 63 pass | `package.json` → `test` |
| 快速单测（`pnpm test:fast`） | 58 | 58 pass | `package.json` → `test:fast` |
| E2E | 3 | 3 pass，8676 ms | 见下表 |

**加载器（同次 `pnpm test` 日志，全仓库参考）**

## 框架优越性（可复现）

| 维度 | 标准 + 实测 |
|------|-------------|
| 基准可审计 | system-Core 基准 + 63 项全量单测；扩展 core 不稀释 11/7/15 |
| CI 速度 | 单测 ~2.5 s |
| 重依赖隔离 | `import` 后 `renderers.size === 0` |
| 测试离线 | `XRK_TEST=1` 跳过远程 MCP |
| 真启动 | E2E 子进程 `bot.run`，探活 12–125 ms |

复现：`pnpm test && pnpm test:e2e`。改标准值须同步 `system-core.mjs` 与本表。
