# 代码与文档审查清单（以当前代码为准）

> 用于发布前自检；与 [AUTH.md](AUTH.md)、[底层架构设计.md](底层架构设计.md) 配套。

## 1. 架构边界

| 检查项 | 期望 | 权威路径 |
|--------|------|----------|
| 业务只在 Core | 无业务逻辑写入 `src/` | `core/<name>/` |
| HTTP API | 继承/导出 `HttpApi` 结构 | `src/infrastructure/http/http.js` |
| 配置三件套 | 默认模板 + schema + 消费代码一致 | `config/default_config/`、`commonconfig/` |

## 2. HTTP 安全

| 检查项 | 期望 |
|--------|------|
| `/api/*` 默认鉴权 | `HttpApi` 自动 `systemAuth`（`src/infrastructure/http/http.js`） |
| 公开接口 | 路由显式 `systemAuth: false` |
| 文件读取 | `InputValidator.validatePath` / `assertPathUnderRoots` |
| 命令执行 | `InputValidator.validateCommand`（如 `tools` 工作流 `run`） |
| 远程访问 | 必须带系统 API Key；仅 `127.*` 回环免 Key |

## 3. 已清理的冗余（维护时注意勿回退）

- 各 `http/*.js` 内重复的 `ensureSystemCoreAuth` → `src/infrastructure/http/auth.js`
- `src/infrastructure/http/utils/helpers.js` 中未使用的中间件工厂已移除
- 废弃 `server.uiCookie` schema 与未使用的 `renderNetworkInfo` 首页面板
- `group.bannedWords` 仅读嵌套字段（`group.yaml`），不再支持扁平 `banned_words_*`
- `chatbot.private` / `chatbot.auto` 与 YAML 字段名一致

## 5. 配置三件套对照（以代码为准）

| 配置名 | 默认模板 | Schema | 存储位置 |
|--------|----------|--------|----------|
| agt | `config/default_config/agt.yaml` | `system.js` 子段 | 全局 `data/server_bots/agt.yaml` |
| device / notice / redis / mongodb / monitor | ✓ | `system.js` | 全局 |
| server / chatbot / group / aistream | ✓ | `system.js` | `data/server_bots/{port}/` |
| `*_llm` 等工厂 | ✓ | `commonconfig/<name>.js` | 随端口 |

API Key：`server.auth.apiKey.file`，默认 `config/server_config/api_key.json`。
