# 代码审查清单

发布前与 [文档审查清单.md](文档审查清单.md) 一并使用。自动化数据以 [框架测试指南.md](框架测试指南.md) 为准。

## 1. 自动化

```bash
pnpm test        # 59 pass，~2.5s（仅 system-Core 基准）
pnpm test:e2e    # 3 pass
```

## 2. 架构

| 检查项 | 期望 |
|--------|------|
| 业务在 `core/<name>/` | 不写 `src/` 业务 |
| HTTP / 插件 / 工作流 | 继承对应基类 |
| Tasker | 模块顶层不 `Bot.tasker.push` |
| 渲染器 | `import` 不启浏览器；截图前 `ensureLoaded()` |
| 框架测试口径 | 仅 `core/system-Core/` 计入 11/7/15/4/3 |

## 3. 安全

| 检查项 | 期望 |
|--------|------|
| `/api/*` | 默认鉴权；`systemAuth: false` 才公开 |
| 文件路径 | `InputValidator` |

## 4. 配置三件套

`config/default_config/*.yaml` + commonconfig schema + 消费代码。文档同步见 [文档审查清单 §C](文档审查清单.md#c-配置与-api-文档)。

## 5. 文档（与代码一致）

勾选 [文档审查清单.md](文档审查清单.md) 全文，至少完成：**关键数字**、**互链**、**变更同步表** 三节。

## 6. 启动冒烟

```bash
node app
```

打开 `http://127.0.0.1:<port>/xrk`，确认截图/渲染可用。
