# XRK-AGT 底层架构设计

> **项目定位**：融合**智能体业务逻辑**的**通用后端** — Runtime（`src/`）提供 HTTP/WS、加载器、工厂与数据库；智能体与业务在 **Core**（`core/*/`）通过 plugin / http / stream / tasker 扩展。  
> 分层边界与配置/AI 链路的单一事实源。专题索引见 [docs/README.md](README.md)。  
> 适用范围：`src/`、`core/*/`、`config/`、`data/server_bots/{port}/`。

| 层 | 路径 | 职责 |
|----|------|------|
| Runtime | `src/bot.js` | HTTP/WS、全局 `Bot`、中间件与路由入口 |
| Infrastructure | `src/infrastructure/`、`src/utils/`、`src/factory/` | Loader、基类、工厂、通用工具（不写业务） |
| Core | `core/<name>/` | `plugin/`、`http/`、`stream/`、`tasker/`、`events/`、`commonconfig/`、`www/<app>/` |

### 工具模块（`src/utils/` 等）

| 模块 | 职责 |
|------|------|
| `paths.js` | `getCoreDirs`、`getCoreSubDirs`、`warmupCoreLayout` |
| `core-fs.js` | `scanFiles`、`statDirs`/`statFiles`、`matchEventPattern` |
| `file-loader.js` | `FileLoader.getCoreSubDirFiles` |
| `loader-constants.js` | `SCAN_IGNORE_PREFIXES`、`LOADER_BATCH_SIZE` 等 |
| `config-yaml.js` | YAML 候选路径与批量读取 |
| `infrastructure/config/config-seed.js` | 配置模板复制、`seedPortConfigs` |
| `http-utils.js` | `HttpResponse` 统一 HTTP/SSE/MCP 响应 |
| `normalize-error.js` | `Error.isError` 统一归一化 |
| `exec-async.js` | Promise 版 `exec`/`execFile`（26.2 单点 promisify） |
| `runtime-globals.js` | `setRuntimeGlobal` / `getRuntimeGlobal`；`isShuttingDown` / `setShuttingDown`；进程标志读写 |
| `bootstrap.js` / `bootstrap-deps.js` | 引导：环境校验、pnpm 依赖安装、Playwright 状态/安装 API |
| `simple-logger.js` | 引导与 restart 菜单的零依赖文件日志 |
| `process-signals.js` | Ctrl+C 三击语义、`registerShutdownHook` |
| `win-utf8.js` | Windows 控制台 UTF-8（菜单与 pino 共用） |
| `db-connect-utils.js` | Redis/MongoDB 共用：`execCommandResult`、`maskConnectionUrl`、`detectArm64`、`connectWithRetry`、`finalizeDbConnectionFailure` |

**工厂**：`src/factory/BaseFactory.js` 提供 `registerProvider` / `listProviders`；ASR/TTS 由 `createMediaFactoryClass()` 生成。LLM 逻辑在 `LLMFactory.js`。

**渲染器**：`src/infrastructure/renderer/browser-renderer-base.js` 为 Puppeteer/Playwright 共用基类。

**数据库**：Redis / MongoDB 见 [database.md](database.md)（`src/infrastructure/database/`、`redis.js`、`mongodb.js`）。

**写法与性能**：[coding-style.md](coding-style.md)

## 3. 配置

- 模板：`config/default_config/*.yaml`
- 运行时：`data/server_bots/{port}/*.yaml`（全局段在 `data/server_bots/*.yaml`）
- LLM 优先级：`请求参数 > provider 配置 > 工作流默认 > 系统默认`
- `/api/v3/chat/completions` 的 `model` 为 **provider key**；真实模型名在 provider YAML 的 `model` / `chatModel`

路径与 schema 变更清单见 skill `xrk-config`、`docs/config-base.md`。

---

## 4. HTTP 鉴权

`/api/` 路由默认 `Bot.checkApiAuthorization`（`http.js` → `auth.js`）；公开接口设 `systemAuth: false`。业务 handler 不重复实现密钥比对。详见 [AUTH.md](AUTH.md)。

---

## 5. 启动

`node app` → `app.js` → `src/utils/bootstrap.js` → `start.js` → `src/bot.js`。步骤、环境变量、Playwright 与 Ctrl+C 语义见 **[startup.md](startup.md)**；Bot 生命周期见 [bot.md](bot.md)。
