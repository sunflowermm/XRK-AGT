# Skills 维护规范

本目录用于存放 Cursor 项目级 Skill，目标是让 Skill 可检索、可执行、可维护。

## 标准结构（必须）

每个 `SKILL.md` 必须包含：

1. YAML Frontmatter
   - `name`: 与目录名一致（推荐 kebab-case）
   - `description`: 一句话描述“何时使用”
2. 固定章节
   - `## 权威入口`
   - `## 适用场景`
   - `## 非适用场景`
   - `## 执行步骤`
   - `## 常见陷阱`

## 禁项（必须删除）

- 操作系统依赖安装教程（如 `brew install`、`apt install`、`winget install`）。
- 与仓库无关的大段通用 Prompt 文案。
- 越权内容（例如要求直接改运行时代码但未给出具体路径与边界）。
- 含糊定位（只说“在某处修改”而不提供路径）。

## 写作要求

- 默认简体中文，结论先行，短句表达。
- 涉及代码定位必须给出明确路径（文件或目录）。
- 文档与实现冲突时，以代码为准，并在 Skill 中标注冲突点。
- 示例尽量给最小可执行片段，避免长篇复制。

## 索引与职责

- 技能索引见 `SKILL_INDEX.md`。
- 新增 Skill 时必须同步更新索引中的“职责边界”和“触发条件”。

## 校验

运行以下命令检查 Skill 规范：

```bash
node scripts/validate-skills.mjs
```

校验内容包括：
- Frontmatter 必填字段
- 固定章节存在性
- 禁用模式命中
- `name` 与目录名一致性
